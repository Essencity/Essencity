### 状态管理方案

采用 Vue 3 Composition API 的 `ref` 和 `reactive` 进行组件级状态管理：

- **用户状态**：登录用户信息、Token 存储在 localStorage
- **视图状态**：通过 `currentView` 控制页面切换
- **数据状态**：各组件使用 `ref` 管理 API 数据

### API 请求处理

通过 fetch API 配合 Vite 代理实现：

```javascript
// Vite 代理配置
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

## 遇到的问题和解决

### 1. 瀑布流布局的高度问题

**问题：** 瀑布流中各卡片高度不一，导致列间高度差异大，影响整体美观。

**解决：** 采用 CSS Flexbox 实现固定宽度列布局，通过 `break-inside: avoid` 控制卡片不被截断，卡片之间设置固定间距 `20px`，使整体布局整齐有序。

### 2. 后端 API 字段命名不一致

**问题：** 后端返回数据使用下划线命名（如 `created_at`），前端习惯使用驼峰命名（如 `createdAt`）。

**解决：** 在 App.vue 中封装 `fixUrl` 和 `getMediaUrl` 函数，统一处理 URL 转换和数据格式化；在获取数据后进行字段映射转换。

### 3. 视频封面上传问题

**问题：** 视频封面需要用户手动上传，但用户体验不佳；自动截取的视频帧可能不是理想封面。

**解决：** 在 CreationPage 中实现双模式封面选择：自动截取视频某一帧作为默认封面，同时支持用户手动上传自定义封面。

### 4. 登录状态持久化

**问题：** 页面刷新后用户登录状态丢失，需要重新登录。

**解决：** 在 App.vue 的 `onMounted` 钩子中从 localStorage 读取用户信息，实现登录状态持久化；同时处理旧版本数据格式兼容问题。

## 心得体会

通过本次前端架构开发，我有以下几点收获：

1. **Vue 3 Composition API 的实践**：相比 Vue 2 的 Options API，Composition API 提供了更灵活的状态逻辑复用方式。通过 `ref`、`reactive`、`watch` 等 API，能够更清晰地组织组件逻辑。

2. **前后端协作的接口约定**：在开发过程中体会到，提前约定好 API 接口格式（统一响应结构、命名规范）能够大幅减少前后端的沟通成本。

3. **组件化开发的优势**：将页面拆分为独立组件（如 MasonryGrid、PostCard）后，不仅代码复用性提高，调试和修改也更加方便。

4. **用户体验细节**：在实现评论功能、关注功能时，体会到好的交互反馈（如按钮状态变化、加载提示）对于用户体验的重要性。

5. **代理配置的重要性**：通过 Vite 的代理配置解决跨域问题，同时统一了 API 请求路径，使得开发环境和生产环境的切换更加平滑。

这次实践让我对前端工程化有了更深入的认识，也为后续的 AI 功能集成（语音搜索）打下了基础。
