1. 后端架构设计与实现

-------

### 1.1 项目技术选型

本人负责后端项目的整体技术架构设计与实现工作。选用 Spring Boot 3 作为后端框架，结合 Spring Data JPA 进行数据持久化操作，使用 MySQL 8.0 作为数据库存储。这一技术栈能够满足高并发访问需求，同时保证系统的可扩展性与维护性。

### 1.2 数据库设计

根据系统需求设计了完整的数据库表结构，主要包括以下实体：

* **用户表 (User)**

  存储用户基本信息，包含用户名、密码、昵称、头像、个人简介、性别等字段。

* **帖子表 (Post)**

  存储用户发布的笔记内容，包含标题、描述、内容类型（图片/视频）、资源URL、封面URL、作者关联、标签等字段。

* **评论表 (Comment)**

  支持二级评论功能，存储评论内容、关联帖子、评论者、父子评论关系等。

* **点赞表 (Like)**

  记录用户对帖子的点赞操作。

* **收藏表 (Collection)**

  记录用户对帖子的收藏操作。

* **关注表 (Follow)**

  实现用户之间的关注关系。

遇到的问题和解决
--------

**问题一：**
在实现用户关注功能时，需要频繁查询用户之间的关系，使用 JPA 原生查询效率较低。

**解决：**
通过在 FollowRepository 中自定义查询方法，使用 Spring Data JPA 的派生查询方式，提高查询效率。

**问题二：**
前后端分离部署时，跨域请求导致 Cookie 无法携带。

**解决：**
在 WebConfig 中配置 `allowCredentials(true)` 并设置合理的 `allowedOrigins`，解决跨域认证问题。
